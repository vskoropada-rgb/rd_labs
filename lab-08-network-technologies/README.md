## Завдання 1. Створення мережевих з’єднань за допомогою netcat

### 1. Встановлення netcat
На обох віртуальних машинах:
```bash
sudo apt update
sudo apt install -y netcat-openbsd

### 2. TCP-з’єднання
Сервер (192.168.56.10)
	nc -l -p 5000

Клієнт (192.168.56.11)
	nc 192.168.56.10 5000

### 3. UDP-з’єднання
Сервер (192.168.56.10)
	nc -u -l -p 5001

Клієнт (192.168.56.11)
	nc -u 192.168.56.10 5001

### 4. Перевірка роботи firewall
sudo ufw enable
sudo ufw deny 5000/tcp
sudo ufw deny 5001/udp
sudo ufw status

Результат
	•TCP: з’єднання завершується помилкою або тайм-аутом
	•UDP: пакети не доставляються без повідомлення про помилку



## Завдання 2. Конфігурація DNS

###  1. Встановлення dnsmasq
sudo apt install -y dnsmasq

### 2. Налаштування DNS
sudo nano /etc/dnsmasq.d/lab.conf
	interface=eth0
	bind-interfaces

	address=/app.lab/192.168.56.100
	address=/db.lab/192.168.56.101
Перезапуск сервісу:
sudo systemctl restart dnsmasq
sudo systemctl status dnsmasq

### 3. Перевірка з клієнта

dig @192.168.56.10 app.lab
nslookup db.lab 192.168.56.10
